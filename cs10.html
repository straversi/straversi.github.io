A Basic Mystery

Evening, Sherlock. I've got a case for you: We need to figure out what the following algorithm does.

Good luck!

The algorithm reports the reverse of a given list.
Let's break it down:
1. Two variables, a and index, are created. a is an empty list, and index is a number equal to the length of the input list.
2. For the length of the list, the item at index index of the input list is added to a. Then, the variable index is decremented by one. This accomplishes adding items of the input list into a backwards!
3. The list a is reported, and it is equal in value to the input list. It is not, however, the same list! This means that changes to one list do not affect the other.

Complexity 1

Let's determine the algorithmic complexity of the following script:

  1. What is the alogrithmic complexity in terms of N, where N is the value of the variable i?
  2. As an bonus test of knowledge, rank the algorithmic complexities from best to worst: In no particular order: O(n), O(n^2), O(log n), O(1), O(2^n), O(n log n).

The complexity of the algorithm is O(log N). Think of it this way: You have to DOUBLE the value of i to get the script to say just ONE more item/take ONE more step (Notice how i is cut in half with each step). This is logarithmic behavior.
Fastest --> O(1) -> O(log n) -> O(n) -> O(n log n) -> O(n^2) -> O(2^n) --> Slowest

Mystery Utility

Another case to solve: pictured below is a list utility. It appears to be recursive. input is a list.

  1. Identify the base case. What is happening?
  2. Identify the recursive case. What is going on?
  3. Finally, what is the function doing?

1. The base case is: If the length of the list is 0, return 0.
2. The recursive case is the "else" case. It returns the result of adding 1 to the result of this function with all items but the first of the list.
3. All together, this function returns the length of the input list!
