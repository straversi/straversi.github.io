My Jekyll Setup

Nov 22, 2015

The permalink format for posts on this site is :categories/:title. That way I can direct students to steven.codes/cs10 for easy access to all cs10 resources.

Organizing Assets

I prefer to structure assets similarly to posts. For example, if there is a post at /cs10/2048-merge, then I’d like the corresponding assets to be stored in /assets/posts/cs10/2048-merge. This means I have to go through this path every time I’d like to include a resource. Asset path tag is a plugin that I use to manage these relative paths. Plugins

Lazy Tweet Embedding automatically converts twitter states urls to twitter cards. For example, if I concatenate https://twitter.com/, StevenTraversi/status/, and a tweet id such as 558448694731608064, the plugin will generate this for me:

Studying machine structures in Soda, "home of top shelf computer scientists". #61C #UCBerkeley #twoscomplement #passbyvalue

— Steven Traversi (@StevenTraversi) January 23, 2015

Clean and easy.

jekyll-compress-html is a top-level layout that I make default.html conform to. Therefore, every HTML file is compressed before being dumped into _site. If I want to forgo compression for a particular page, I add tags: no_compress to that page’s front matter. This can be useful for pages that have lots of // style javascript comments, as jekyll-compress-html freaks out when it sees those.

Disqus is a forum drop-in that I use at the bottom of each article to create a comments section. It’s powered by a script in _layouts/post.html, but must be explicitly enabled in the yaml of each article with comments: true.

Highlight_plus. Hey, I wrote this! It’s an extension of Jekyll’s default syntax highlighter. This one allows the language name or file name to be printed above a code block. ◼
